## 八、假设与决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| chat.service.ts 类型 | 统一用 ChatChunk（非 StreamChunk 别名） | 消除历史命名分歧，shared 已导出 ChatChunk |
| ChatPage SSE | 改用 chatStream() 封装 | 修复 404 bug + JWT 注入 + 消除硬编码 |
| SettingsPage 空状态 | `editingId === null` | 原逻辑反了，无配置且非编辑模式才显示提示 |
| schema enum | Mode + MessageRole | Prisma enum 约束 + 应用层校验双保险 |
| prisma migrate | 优先 generate，migrate 可选 | 数据库可能不可用，不阻塞类型生成 |
| Session/Message | 创建后端模块 + 前端迁移 | 替代 localStorage，实现多端同步 |
| ChatPage 拆分 | 5 store + 3 hooks + 6 memo 组件 | 目标 <300 行 |
| 提示词 | prompts/ 包模块化 + 工厂函数 | 替代单文件常量，支持按 mode 动态选择 |
| CI 版本 | pnpm 11.9 + Python 3.14 + Node 22 | 与本地一致，2026 LTS |
| changesets | 初始化 + release.yml | 自动化版本管理 |

**风险**:
- ChatPage 拆分需保持 SSE 流式逻辑正确性（最复杂步骤，需逐一迁移 processChunk 逻辑）
- `prisma migrate dev` 需数据库可用，若不可用则仅 `prisma generate`
- Phase D 工作量最大（5 store + 3 hooks + 6 组件 + 路由），建议分小步验证
