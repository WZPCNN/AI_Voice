## 3. 提议的变更（Proposed Changes）

### Phase D：ChatPage 拆分 + Sidebar 重复修复

#### D1. 创建 3 个 Hooks

**新建 `apps/web/src/hooks/useChat.ts`**
- 封装 `sendMessage`、`stopGeneration`、SSE 流式处理
- 从 `chatStore` 读取 `sessionId/messages/streaming/activeAgents`
- 从 `uiStore` 读取 `modes/images`
- 从 `configStore` 读取 `selectedConfigId`
- 调用 `chatStream()`，每个 chunk 调用 `chatStore.processChunk()`
- 暴露：`{ sendMessage, stopGeneration, streaming }`
- 内部用 `useRef<AbortController>` 管理取消

**新建 `apps/web/src/hooks/useSlashCommands.ts`**
- 封装斜杠命令的过滤、键盘导航、执行
- 从 `uiStore` 读取 `slashQuery/slashVisible/selectedIdx/modes`
- 暴露：`{ filteredCmds, executeCommand, handleKeyDown }`
- `filteredCmds` 用 `useMemo` 根据 `slashQuery` 过滤 `COMMANDS`

**新建 `apps/web/src/hooks/useConversations.ts`**
- 封装会话列表管理
- 从 `sessionStore` 读取 `conversations/currentSessionId`
- 从 `chatStore` 读取 `sessionId/messages`
- 暴露：`{ conversations, currentSessionId, newSession, selectSession, deleteSession, loadMessages }`
- `selectSession` 内部调用 `api.sessions.messages(id)` 加载历史消息到 chatStore

#### D2. 创建 5 个 chat 子组件

**新建 `apps/web/src/chat/MessageList.tsx`**（`React.memo`）
- Props: `messages`, `streaming`, `planMessageIdx`, `planSteps`, `agentPlans`, `collapseAgents`, `onRetry`, `onCopy`
- 渲染：空状态欢迎页 + 错误条 + 消息列表（map MessageBubble）
- 用 `useMemo` 计算是否空状态

**新建 `apps/web/src/chat/MessageBubble.tsx`**（`React.memo`）
- Props: `msg`, `index`, `isLast`, `streaming`, `planMessageIdx`, `planSteps`, `agentPlans`, `collapseAgents`, `onRetry`, `onCopy`
- 渲染：消息气泡（user/assistant 对齐）、Agent 标记、图片附件、PlanSection（仅当前 plan 消息）、Markdown 内容、中断标记 + 重试、复制按钮、工具调用

**新建 `apps/web/src/chat/ChatInput.tsx`**（`React.memo`）

- 渲染：图片预览行 + Tiptap 编辑器 + 模式徽章 + 文件上传按钮 + ModelSelector + SlashCommandPalette + 发送/停止按钮
- 内部用 `useRef<HTMLInputElement>` 管理文件 input

- Props: `selectedConfig`, `configs`, `open`, `onToggle`, `onSelect`, `onAddModel`

#### D3. 重构 ChatPage.tsx（1444 行 → 目标 < 300 行）

**修改 `apps/web/src/pages/ChatPage.tsx`**

      <main>
        <ChatHeader ... />

        <ChatInput ... />
      </main>
    )
  }
