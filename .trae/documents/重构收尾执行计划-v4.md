## 四、Phase G：skills + mcp 完整实现

### 设计概述

**Skills（核心能力系统）**:
- 3 个内置技能: `code-review`（代码审查）、`summarize`（文档摘要）、`web-search`（网络搜索）
- 每个技能 = 专属系统提示词 + 专属工具集
- skills 模式: 用户选择一个技能 → Agent 以该技能的提示词+工具执行
- 技能为内置定义（Python 端注册表），无需数据库 CRUD

**MCP（Model Context Protocol 控制面板）**:
- 用户可配置多个 MCP 服务器（stdio/SSE 传输）
- mcp 模式: Agent 连接用户配置的 MCP 服务器 → 发现工具 → 使用这些工具执行
- MCP 服务器配置存储在数据库（McpServer 表）
- Python 端使用 `mcp` 包连接服务器，将 MCP 工具包装为 LangChain 工具

`agents/src/mcp_client/manager.py`:
```python
"""MCP 工具加载器 — 连接 MCP 服务器,将工具包装为 LangChain Tool。"""
import json
from typing import Any
from langchain_core.tools import StructuredTool
from pydantic import BaseModel, create_model

async def load_mcp_tools(servers: list[dict]) -> list:
    """连接多个 MCP 服务器,返回包装后的 LangChain 工具列表。

    Args:
        servers: MCP 服务器配置列表 [{transport, command/url, env}]
    Returns:
        LangChain Tool 列表
    """
    try:
        from mcp import ClientSession, StdioServerParameters
        from mcp.client.stdio import stdio_client
        from mcp.client.sse import sse_client
    except ImportError:
        return []

    tools = []
    for srv in servers:
        try:
            session_tools = await _connect_and_discover(srv)
            tools.extend(session_tools)
        except Exception:
            continue
    return tools

async def _connect_and_discover(server: dict) -> list:
    """连接单个 MCP 服务器,发现工具并包装。"""
    from mcp import ClientSession

    transport = server.get("transport", "stdio")
    if transport == "stdio":
        from mcp.client.stdio import stdio_client, StdioServerParameters
        params = StdioServerParameters(
            command=server.get("command", ""),
            args=server.get("command", "").split()[1:] if server.get("command") else [],
            env=server.get("env"),
        )
        async with stdio_client(params) as (read, write):
            async with ClientSession(read, write) as session:
                await session.initialize()
                result = await session.list_tools()
                return [_wrap_tool(session, t, server.get("name", "")) for t in result.tools]
    elif transport == "sse":
        from mcp.client.sse import sse_client
        async with sse_client(server.get("url", "")) as (read, write):
            async with ClientSession(read, write) as session:
                await session.initialize()
                result = await session.list_tools()
                return [_wrap_tool(session, t, server.get("name", "")) for t in result.tools]
    return []

def _wrap_tool(session, mcp_tool, server_name: str):
    """将 MCP 工具包装为 LangChain StructuredTool。"""
    from langchain_core.tools import StructuredTool

    async def _call(**kwargs):
        result = await session.call_tool(mcp_tool.name, kwargs)
        return result.content if hasattr(result, "content") else str(result)

    return StructuredTool.from_function(
        coroutine=_call,
        name=f"mcp_{server_name}_{mcp_tool.name}",
        description=mcp_tool.description or f"MCP tool: {mcp_tool.name}",
    )
```

```python
            temperature=temperature, api_key=api_key, base_url=base_url,
            tool_names=[t.name for t in skill.tools],
        )
        # 使用技能专属提示词
        async for chunk in runner.run(session_id, content, history=history,
                                       system_prompt=skill.system_prompt):
            await redis.rpush(f"agent:response:list:{session_id}", chunk)
    else:
        # 技能不存在,回退到 exec
        async for chunk in runner.run(session_id, content, history=history):
            await redis.rpush(f"agent:response:list:{session_id}", chunk)

elif mode == "mcp":
    # mcp 模式:连接 MCP 服务器,加载工具
    mcp_servers = request.get("mcp_servers", [])
    mcp_tools = await load_mcp_tools(mcp_servers)
    # 创建带 MCP 工具的 runner
    runner = AgentRunner(
        model_provider=provider, model_name=model,
        temperature=temperature, api_key=api_key, base_url=base_url,
        tool_names=None,  # None = 所有内置工具
    )
    # 将 MCP 工具加入 runner 的工具列表
    runner.tools = runner.tools + mcp_tools
    runner.llm = runner.llm.bind_tools(runner.tools)
    async for chunk in runner.run(session_id, content, history=history):
        await redis.rpush(f"agent:response:list:{session_id}", chunk)
```

## 六、假设与决策表

| 编号 | 决策 | 理由 |
|------|------|------|
| A1 | skills 为内置定义,无数据库 CRUD | 技能数量少且固定,避免过度工程化 |
| A2 | MCP 服务器配置存数据库 | 用户自定义配置需要持久化 |
| A3 | MCP 使用 `mcp` Python 包 | 官方协议实现,成熟可靠 |
| A4 | skills 模式复用 AgentRunner | 技能 = 提示词 + 工具子集,无需新 runner 类 |
| A5 | mcp 模式复用 AgentRunner + 动态工具 | MCP 工具包装为 LangChain Tool 后注入 runner |
| A6 | prompts 拆为 4 文件包 | identity/capabilities/tools/system 各自独立,可单独引用 |
| A7 | runner.py 引用不变 | `__init__.py` re-export SYSTEM_PROMPT,导入路径兼容 |
| A8 | MCP 连接为 per-request | 简单,无状态管理,适合当前规模 |
| A9 | SettingsPage 增加 Tab | 复用现有页面结构,避免新增路由 |
| A10 | ci.yml Node 22 + pnpm 11 | 与本地开发环境一致 |
| A11 | skills 端点硬编码 3 个 | 与 Python 端 BUILTIN_SKILLS 对应,无需同步机制 |
| A12 | Prisma 迁移需手动执行 | schema 变更后 `prisma migrate dev` |
